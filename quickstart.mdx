## Setup your development

<Warning>You will first need to have node and npm/yarn installed</Warning>

### Step 1. Unzip your downloaded folder:

- Mac - double click to unzip your file
- Windows - extract zip files

### Step 2. Open the unzipped template in your code editor:

VSCode documentation can be found [here](https://learn.microsoft.com/en-us/visualstudio/ide/develop-code-in-visual-studio-without-projects-or-solutions?view=vs-2022)

### Step 3. Install your dependencies

Open up your terminal in your code editor and install your dependencies.

<CodeGroup>

```bash npm
npm install
```

```bash yarn
yarn install
```

</CodeGroup>

### Step 4. Install the sanity cli
<CodeGroup>

```bash npm
# Running the CLI without global installation
npx -y sanity@3 [command]


# Alternatively installing the CLI globally
npm install --global sanity@3
```

```bash yarn
# Running the CLI without global installation
npx -y sanity@3 [command]


# Alternatively installing the CLI globally
yarn global add sanity@3
pnpm install --global sanity@3
```

</CodeGroup>

### Step 5. Create a sanity project

```bash
npx create sanity@3 init --env
```

After running the command, you will be prompted to configure your project:
- Create a new new project
- Set your project name: i.e. `My Portfolio`
- Use Use the default dataset configuration? `Yes`
- Would you like to add configuration files for a Sanity project in this Next.js folder? `No`

<Warning>Ensure that you select `no` when asked if you would like add config files for your Sanity project</Warning>

### Step 6. Create sanity api read token

- Copy `SANITY_API_READ_TOKEN` from `.env.lcoal.example` file into the `.env` file
- Navigate to your [Sanity Projects](https://www.sanity.io/manage/personal/projects) and click on your new project
- Click on the API tab and then Tokens
- Create a new API token, i.e. `viewer`
- Copy the API token generated and set it to the  `SANITY_API_READ_TOKEN` in your `.env` file.
